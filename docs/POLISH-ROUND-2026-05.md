## Правки

### Редактор задач (`TaskRow.tsx`)
- `<input>` → `<textarea>` (auto-grow, длинный текст переносится).
- Курсор при входе в правку — **в конец**, не выделять весь текст.
- Caret видимый (sage), был белый/невидимый на телефоне.
- Кнопка `×` очистки текста (фокус сохраняется), видна пока есть текст.
- `⋮`-меню **видно всегда** (было на ховере) + hit-area **44×44** (Apple HIG).
- Шрифт пункта 15 → **16px**, строка чуть выше.

### Лента — шапка (`Mysli.tsx`)
- **Только строка поиска + колокольчик** — sticky (флоатит вверху при скролле),
  frosted-фон. Чипы-фильтры и view-segment — в обычном потоке, скроллятся под неё.
- Gap сверху: `safe-area-inset-top + 16px`.

### Поиск (`SearchBar` в `atoms.tsx`)
- Крупнее: padding 13 → **18px (18×20)**, шрифт 15 → **17px**, иконка 18 → 20.

### Нижняя навигация (`Nav.tsx`)
- Компактнее: nav-иконки 22 → 20, лейблы 11 → 10, FAB «создать» 56 → **48**.

### Лента — bare-link (`adapters.ts`, прошлый заход)
- Голая ссылка: заголовок = host (не длинный URL), без дубля src-пилюли,
  пустой preview когда `raw_text == url`.

## 🔵 BACKEND — что требует работы на бэке

### 1. Контекст ссылки (бед `z9q`) — отдельный PRD
**Файл:** монорепо `docs/prd/LINK-CONTEXT-EXTRACTION.md`.
Суть: голая ссылка показывает только host + вечный плейсхолдер «Brain читает
ссылку…», потому что бэк не вытаскивает контент по URL. Нужен fetch + OG/meta
extract + SSRF-защита. Фронт уже рендерит `summary`/`key_ideas`/`title` когда
заполнены — контракт менять не надо.
