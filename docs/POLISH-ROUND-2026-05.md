## Правки

### Редактор задач (`TaskRow.tsx`)
- `<input>` → `<textarea>` (auto-grow, длинный текст переносится).
- Курсор при входе в правку — **в конец**, не выделять весь текст.
- Caret видимый (sage), был белый/невидимый на телефоне.
- Кнопка `×` очистки текста (фокус сохраняется), видна пока есть текст.
- `⋮`-меню **видно всегда** (было на ховере) + hit-area **44×44** (Apple HIG).
- Шрифт пункта 15 → **16px**, строка чуть выше.

### Лента — шапка (`Mysli.tsx`)
- **Только строка поиска + колокольчик** — sticky (флоатит вверху при скролле),
  frosted-фон. Чипы-фильтры и view-segment — в обычном потоке, скроллятся под неё.
- Gap сверху: `safe-area-inset-top + 16px`.
- Колокольчик — **квадрат 56×56** (= высоте поиска), радиус как у поиска (был круг 46).

### Sticky-фикс (почему шапка сначала НЕ держалась) — `layout.css`, `App.tsx`
Две причины ломали `position:sticky`, обе устранены:
1. `overflow-x: hidden` на `.app-shell` / App-root создавал скролл-контейнер →
   заменено на **`overflow-x: clip`** (клипает по горизонтали, контейнер не создаёт).
2. `.screen-fade` имел `animation … both` → после анимации держался
   `transform: translateY(0)`; **любой transform на родителе ломает sticky** →
   убран `fill-mode` (элемент возвращается к `transform:none`).

### Поиск (`SearchBar` в `atoms.tsx`)
- Крупнее: шрифт 15 → **17px**, иконка 18 → 20.
- Высота фиксированная **56px** (вместо vertical-padding) — предсказуемый матч с bell.

### Шторка напоминания (`ReminderPickerSheet`, `Calendar`, `sheetPrimitives`)
- Прошлые даты в календаре **задизейблены** (line-through, не кликабельны); навигация
  в прошлые месяцы заблокирована.
- Кнопка «напомнить» недоступна для времени **в прошлом** (раньше включалась) +
  tooltip/hint почему.
- «выбрать дату и время» — теперь **кнопка** (не radio), раскрывает календарь;
  внутри — кнопка **«назад»** к пресетам.
- Выбранное время — в **слитном заголовке** «напомнить <когда>» (UI-шрифт, не mono).
- Пресеты — **карточки 2 в ряд** (вид отличается от текста напоминания).
- Текст напоминания — **редактируемый input**, не приглушён.
- Открытие на **существующем времени** (`initialISO`): snooze 9:00 → пикер на 9:00.

### Шторки напоминаний — раунд 4
- **Список:** × унифицирован (общий `SheetCloseBtn`); счётчик → **бейдж** (sage-пилюля);
  время **над текстом**; sentence-case («Напоминания», «Сегодня»).
- **Пикер:** при переносе 1-я карточка = исходное время (выбрана); кнопка **disabled
  пока не изменишь** время/текст (timestamp-сравнение); время **над текстом**;
  текст — **textarea как пункт списка** (консистентность с TaskRow); время+текст+кнопка
  закреплены, скроллится только календарь+барабан.
- **Календарь/барабан:** `Calendar compact` (ячейки 30px) + `TimeWheel compact` (3 строки) —
  чтобы барабан времени влезал под календарём.

### Барабан-поповер времени (тикет `tb1`, 2026-06-01)
В custom-режиме пикера (`ReminderPickerSheet`, кнопка «Выбрать дату и время») раньше
показывалась стопка `Calendar compact` + `TimeWheel compact` — тесно (прошлый фидбек
«слишком напихано»). Теперь:
- Календарь (full compact) + **строка «Время»** с пилюлей-значением (`HH:MM`, mono, sage-tint).
- Тап по строке → **поповер**: `position:fixed` overlay (z-index 110, над шторкой z101),
  карточка с `TimeWheel` (полный, 5 строк) + кнопка «Готово». Тап-вне (бэкдроп) или
  «Готово» закрывают; `hour`/`minute` обновляются live → бейдж времени в заголовке и
  пилюля в строке синхронны.
- Состояние `timeOpen` локально в компоненте. `TimeWheel` больше не `compact` здесь
  (в поповере есть место под 5 строк).

### ✅ Глобальная типографика sentence-case (2026-06-01)
Product-override залоченного на lowercase DS. Первая буква заглавная по **всему** UI:
- **Видимые лейблы:** Nav (Мысли/Пространства/Я); заголовки шторок (Дедлайн, В пространство,

### Унификация навигации шторок (правило `‹` vs `×`)
Единая конвенция (см. `CLAUDE.md` → «Конвенция навигации шторок»):
- **Назад** `‹` (шеврон, слева, `onBack`) — шаг назад в флоу.
- **Закрыть** `×` (справа, `onClose`) — закрыть целиком; свайп/клик-вне тоже.
- Точка входа — только `×`; вложенная — `‹`+`×`; под-состояние — `‹` без `×`.

Реализация: `SheetTitle` получил `onBack`; новый `SheetBackBtn` + icon `chevronLeft`.
Применено: ActionSheet (`×`), MoveToSpace (`‹`+`×`), QuickCreate (`×`),
ReminderPicker (пресеты `‹`+`×`, календарь `‹`), RemindersSheet (`×`).

## 🔵 BACKEND — что требует работы на бэке

### 1. Контекст ссылки (бед `z9q`) — отдельный PRD
**Файл:** монорепо `docs/prd/LINK-CONTEXT-EXTRACTION.md`.
Суть: голая ссылка показывает только host + вечный плейсхолдер «Brain читает
ссылку…», потому что бэк не вытаскивает контент по URL. Нужен fetch + OG/meta
extract + SSRF-защита. Фронт уже рендерит `summary`/`key_ideas`/`title` когда
заполнены — контракт менять не надо.

### 2. Правка текста напоминания (тикет `8uu`) — бриф
**Файл:** монорепо `docs/prd/REMINDER-TEXT-EDIT.md`.
Суть: `PATCH /reminders/{id}` принимает только `fire_at`, текст в `payload["text"]`
не меняется → на переносе текст read-only (иначе молча терялся бы). Нужно принять
опциональный `text` и персистить в payload (⚠️ JSONB-мутация через копию/flag_modified).
Разблокирует правку текста напоминаний в `ReminderPickerSheet`.

### 3. Правка текста любых заметок (тикет `0rn`) — бриф
**Файл:** монорепо `docs/prd/BOOKMARK-TEXT-EDIT.md`.
Суть: `BookmarkUpdate` не разрешает `raw_text`/`transcription` → нельзя дописать заметку
к ссылке, поправить расшифровку голосового. Нужно принять тело текста + честно отражать
переобработку через **`ai_status`** (фронт показывает «Brain переосмысливает…» и рефетчит).
Глубина переобработки (embed/full/none) — **решение бэка**, фронт подстроится по `ai_status`.
